# 论文深度学习技能 (Paper Deep Learning Skill)

## 概述

系统性阅读和掌握学术论文的全流程技能，核心设计原则：
- **分模块管理**：论文背景、论文内容各章节、涉及基础知识，各自独立成块
- **联网补充知识**：每个模块执行专项搜索，搜索结果和链接在对话中展示，并写入文件
- **就近记录**：每个模块的补充资源链接 + 提问问答，紧跟在该模块下方
- **充分提问**：每个模块至少 2～4 个问题，全面检验该模块的理解

---

## 联网搜索规范 🌐（全局适用）

**这是整个技能最重要的操作规范，每个模块都必须遵守。**

### 搜索执行要求

每个模块在讲解后、提问前，**必须执行联网搜索**，不得跳过：
1. 执行 `web_search` 和必要时的 `web_fetch`（获取完整页面内容）
2. 在对话中向用户展示搜索结果（见下方展示格式）
3. 将资源链接实时写入进度文件对应模块的补充资源区

### 搜索结果对话展示格式

每次搜索后，在对话中用以下格式呈现，让用户看到资源：

```
🌐 **[模块名] 补充资源**

以下是为你找到的相关资料，点击可深入阅读：

📄 **[资源标题1]**
   来源：[网站名/期刊名] | 类型：[博客/论文/文档/视频]
   简介：[1-2句说明这个资源讲了什么，与当前内容的关联]
   🔗 [完整URL]

📄 **[资源标题2]**
   来源：[网站名] | 类型：[类型]
   简介：[简介]
   🔗 [完整URL]

📄 **[资源标题3]**
   ...
```

**要求**：
- 每个模块至少找到 **3 个有效资源**，优先选取：官方文档 > 顶级博客（distill.pub, lilianweng.github.io 等） > arxiv 论文 > 教程视频
- 每个链接必须是完整可点击的 URL，不能是标题或描述
- 简介必须说明"为什么这个资源对理解当前模块有帮助"

### 资源写入文件格式

搜索展示后，立即将以下内容 append 到进度文件对应模块的补充资源表：

```markdown
| [类型] | [资源标题] | [完整URL] | [搜索日期] | [与论文的关联说明] |
```

## STAGE 0：开场 — 检查进度与接收论文

### 0.1 检查已有进度文件（必须第一步执行）
```bash
ls /mnt/user-data/outputs/paper_learning_*.md 2>/dev/null
```

**找到文件** → 用 `view` 读取，向用户展示：
- 论文标题 + 当前所在模块
- 各模块完成状态（checklist）
- 已收录的补充资源数量
- 上次最后一道未完成的问题（若有）
- 询问：继续上次 / 从头开始 / 跳到特定模块

**没有文件** → 进入 0.2

### 0.2 接收论文

| 来源 | 处理 |
|------|------|
| PDF 上传 | 读取 `/mnt/user-data/uploads/` |
| 粘贴文本 | 直接处理 |
| arxiv / DOI 链接 | `web_fetch` 抓取完整内容 |
| 仅标题 | `web_search` 搜索摘要、作者、发表信息，`web_fetch` 获取论文页面 |

### 0.3 确认学习模式
- **完整深度学习**（推荐）：逐模块学习 + 联网补充 + 提问 + 记录
- **快速通读**：只讲核心内容，不提问，搜索资源但简化展示
- **指定模块**：用户指定从哪个模块开始

## STAGE 1：论文背景模块

> 📁 对应进度文件章节：`## 【模块一】论文背景`

### 1.1 背景信息提取与呈现

**执行搜索（必须）**：
```
web_search("[论文标题] author institution year conference")
web_search("[论文标题] semantic scholar citation")
web_fetch  论文的 arxiv / 官方页面（获取 abstract、作者信息）
```

提炼并呈现：
- 标题、作者、机构、发表渠道与级别
- 研究领域和子方向
- 被引用次数（数量级）

**研究背景**（结合搜索结果讲解）：
- 这个研究方向的兴起背景和发展阶段
- 论文发表时领域的研究现状与主流方法
- 论文的核心动机：当时存在什么痛点？
- 这篇论文在领域中的定位

### 1.2 背景知识联网搜索 🌐

**必须执行的搜索**：
```
web_search("[研究领域] history development overview")
web_search("[论文之前的主流方法] what is it explained")
web_search("[论文标题] follow-up work influence")
```

**可选补充搜索**（根据领域特点）：
```
web_search("[研究领域] survey 2023 OR 2024")
web_search("[关键对比方法] tutorial beginner")
```

执行后按「联网搜索规范」格式展示，并写入文件。

### 1.3 背景模块提问（至少 3 题）

提问后将完整问答记录到文件 `【模块一】> 1.4 问答记录`。

**必问**：
- B1：这篇论文是在什么背景下诞生的？当时领域面临的核心困境是什么？
- B2：这篇论文属于什么类型的工作？你怎么判断的？

**选问**（选 1～2 题）：
- B3：在这篇论文出现之前，主流方法是什么？它们的核心局限是什么？
- B4：这篇论文发表渠道说明什么？（顶会/顶刊/arxiv 的含义）
- B5：你认为作者为什么选择在这个时间点做这个工作？

## STAGE 2：论文内容模块（逐子模块分析）

> 📁 对应进度文件章节：`## 【模块二】论文内容`，下设子模块 2A～2E

每个子模块固定流程：**① 讲解 → ② 联网搜索并展示 → ③ 写入资源链接 → ④ 提问并就近记录问答**

---

### 2A：问题定义与动机

**讲解**：核心问题精确表述、对现有方法不足的具体批判、论文核心假设

**联网搜索 🌐**（必须）：
```
web_search("[现有方法名] limitations failure cases")
web_search("[论文问题域] benchmark dataset overview")
web_search("[论文问题] why is it hard challenging")
```
展示资源 → 写入文件 `2A 补充资源`

**提问（至少 3 题）**，问答记录到 `2A 问答记录`：
- A1：论文要解决的问题，一句话精确描述是什么？
- A2：作者批评了哪些现有方法？这些批评有说服力吗？
- A3：如果这个问题不被解决，会有什么实际影响？
- A4（选）：论文的核心假设是什么？在什么情况下会不成立？

---

### 2B：方法与核心创新

**讲解**：方法总体框架、核心创新点（逐条）、与现有方法本质区别、关键设计直觉

**联网搜索 🌐**（必须）：
```
web_search("[论文方法名] explained intuition how it works")
web_search("[论文核心技术] tutorial step by step")
web_search("[对比方法] vs [论文方法] comparison difference")
web_fetch  找到的最权威的解释页面（distill.pub、官方博客等）
```
展示资源 → 写入文件 `2B 补充资源`

**提问（至少 4 题）**，问答记录到 `2B 问答记录`：
- M1：不看原文，用自己的话描述这个方法的核心思路
- M2：最关键的设计决策是什么？为什么这样设计而不是其他方式？
- M3：这个方法和 [对比方法X] 的本质区别是什么？
- M4：如果你来实现这个方法，最难的地方在哪里？
- M5（选）：这个方法有哪些超参数？设置不当会怎样？
- M6（选）：方法背后有什么"优雅"的数学或工程直觉？

---

### 2C：技术细节

**讲解**：关键模块（输入→处理→输出）、重要公式的直觉、算法流程、工程要点

**联网搜索 🌐**（必须）：
```
web_search("[核心公式/技术名] derivation intuition explained")
web_search("[关键模块名] implementation pytorch tensorflow")
web_search("[论文方法] code implementation github")
web_fetch  找到的实现教程或代码仓库页面
```
展示资源 → 写入文件 `2C 补充资源`

**提问（至少 3 题）**，问答记录到 `2C 问答记录`：
- T1：[关键模块] 的输入和输出分别是什么？在整个系统中的作用？
- T2：[核心公式] 中每一项代表什么？为什么这样设计？
- T3：如果去掉 [某关键组件]，整个方法会出什么问题？
- T4（选）：这个方法的计算复杂度是多少？瓶颈在哪里？

---

### 2D：实验与结果

**讲解**：实验设计合理性分析、核心结果解读（重要数字）、Ablation study、反直觉发现

**联网搜索 🌐**（必须）：
```
web_search("[数据集名] benchmark description statistics")
web_search("[评估指标名] what it measures how to calculate")
web_search("[对比基线方法] paper results performance")
```
展示资源 → 写入文件 `2D 补充资源`

**提问（至少 3 题）**，问答记录到 `2D 问答记录`：
- E1：作者选择这些对比基线是否合理？有没有遗漏重要对比？
- E2：哪个实验数据最能说明方法的有效性？为什么？
- E3：Ablation study 中，哪个组件被证明最重要？
- E4（选）：如果你来重新设计实验，会增加哪些实验？

---

### 2E：结论、贡献与局限

**讲解**：作者总结的贡献（对照方法章节）、承认的局限性、未来工作方向、对领域的实际影响

**联网搜索 🌐**（必须）：
```
web_search("[论文标题] criticism limitations analysis")
web_search("[论文方法] weakness failure cases when it fails")
web_search("[论文领域] recent advances after [论文年份]")
```
展示资源 → 写入文件 `2E 补充资源`

**提问（至少 2 题）**，问答记录到 `2E 问答记录`：
- C1：这篇论文最大的局限性是什么？作者是否诚实地承认了？
- C2：在这篇论文基础上，最有价值的后续研究方向是什么？
- C3（选）：这篇论文对你实际工作/研究有什么启发？

## STAGE 3：基础知识模块

> 📁 对应进度文件章节：`## 【模块三】基础知识`，每个概念独立成条

每遇到需要补充的前置概念，立即建立一个知识条目，结构如下：

### 知识条目固定流程
① 讲解（直觉 → 类比 → 正式定义）
② 联网搜索 🌐 并展示资源
③ 写入资源链接到该条目
④ 提问并就近记录问答

**每个条目的联网搜索（必须）**：
```
web_search("[概念名] explained for beginners intuition")
web_search("[概念名] visual explanation example")
web_fetch  找到的最清晰的教程页面
```

**知识分层**：

| 层次 | 说明 | 搜索深度 | 提问要求 |
|------|------|---------|---------|
| 基础层 | 理解论文必须掌握的前置概念 | 深度搜索，≥3个资源 | 必须提问，≥2题 |
| 理解层 | 帮助深入理解方法的进阶知识 | 中度搜索，≥2个资源 | 建议提问，1～2题 |
| 扩展层 | 相关前沿工作，了解即可 | 快速搜索，≥1个资源 | 不强制提问 |

## STAGE 4：综合检验模块

> 📁 对应进度文件章节：`## 【模块四】综合检验`

完成前三个模块后进行，同时做最后一轮**扩展资源搜索**：

**综合扩展搜索 🌐**（必须）：
```
web_search("[论文标题] review summary explained")
web_search("[论文方法] applications real world use cases")
web_search("[研究领域] future directions open problems 2024")
```
展示资源 → 写入文件 `模块四 补充资源`

**综合提问（至少 4 题）**，记录到 `模块四 问答记录`：

整合类：
- S1：用 3 句话概括这篇论文（问题 + 方法 + 意义）
- S2：向非专业朋友解释这篇论文，你会怎么说？

批判类：
- S3：这篇论文最大的优点和缺陷各是什么？
- S4：你相信论文的结论吗？有没有让你存疑的地方？

延伸类（选 1～2 题）：
- S5：这个方法在什么场景下会失效？
- S6：如果给你一周时间改进这个方法，你会做什么？

## 进度文件写入规范 ⭐

### 文件命名
```
/mnt/user-data/outputs/paper_learning_[2~4个英文关键词].md
```

### 写入时机（实时，不积累到最后）

| 事件 | 写入内容 |
|------|---------|
| STAGE 0 完成 | 创建文件，基本信息 + 进度总览框架 |
| 某模块讲解完成 | 追加该模块内容笔记 |
| **联网搜索完成** | **立即追加搜索到的资源链接到对应模块补充资源表** |
| **提出问题** | **立即 append 问题到对应模块问答区** |
| **收到用户回答** | **立即 append 完整回答 + 完整点评** |
| 模块完成 | 更新进度总览 checklist |
| 会话结束 | 更新掌握度 + 下次计划 + 统计资源总数 |

### 问答记录格式（每模块独立追加）

```markdown
#### [编号]｜[🟢Level 1 / 🟡Level 2 / 🔴Level 3]｜[YYYY-MM-DD HH:MM]

**❓ 问题**
[完整问题原文]
```

## 注意事项

1. **搜索必须执行**：每个模块的联网搜索不得跳过，不能以"已有知识"代替
2. **链接必须完整**：文件中的 URL 必须是完整可点击链接，不得截断或省略
3. **展示后写入**：搜索结果先在对话中向用户展示，然后立即写入文件
